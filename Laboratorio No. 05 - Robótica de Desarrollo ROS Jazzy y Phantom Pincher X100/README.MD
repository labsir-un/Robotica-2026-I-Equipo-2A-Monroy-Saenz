# Actividad 7. Movimiento individual de articulaciones

Para realizar el desplazamiento simultĆ”neo de todas las articulaciones se implementaron dos funciones principales: `desplazar_a_configuracion()` y `ejecutar_secuencia_pasos()`. Ambas trabajan de forma conjunta para enviar una secuencia de posiciones al robot utilizando el tĆ³pico de estados articulares de ROS 2.

La funciĆ³n `desplazar_a_configuracion(self, angulos_grados)` es la encargada de publicar una configuraciĆ³n articular completa. 
```python

        """Toma una lista de 5 Ć”ngulos en grados y los publica a RViz."""

        msg.name = self.joint_names

        self.publisher_.publish(msg)
```
Como parĆ”metro recibe una lista con cinco Ć”ngulos, correspondientes a las articulaciones del PhantomX Pincher, expresados en grados. En primer lugar, se crea un mensaje de tipo `JointState`, el cual es el formato estĆ”ndar utilizado por ROS 2 para representar el estado de las articulaciones de un robot. Posteriormente, se actualiza el campo `header.stamp` con la hora actual mediante `self.get_clock().now().to_msg()`, permitiendo que el mensaje quede correctamente sincronizado dentro del sistema ROS.

A continuaciĆ³n, el atributo `msg.name` se inicializa con la lista `self.joint_names`, la cual contiene el nombre de cada una de las articulaciones del robot. Esto permite que cada posiciĆ³n enviada sea asociada a la articulaciĆ³n correspondiente. Posteriormente, la lĆ­nea

```python

```

convierte cada uno de los Ć”ngulos recibidos en grados a radianes utilizando la funciĆ³n `math.radians()`. Esta conversiĆ³n es necesaria porque el sistema de representaciĆ³n de posiciones articulares en ROS 2 utiliza radianes como unidad de medida. Finalmente, el mensaje completo es publicado mediante

```python

```

lo que provoca que todas las posiciones sean enviadas al mismo tiempo y que el robot actualice simultĆ”neamente todas sus articulaciones.

La segunda funciĆ³n, `ejecutar_secuencia_pasos(self)`, tiene como objetivo ejecutar automĆ”ticamente una serie de configuraciones previamente definidas.

Inicialmente, se muestra un mensaje informativo en la consola indicando el comienzo de la actividad mediante:


DespuĆ©s se define la variable `configuraciones`, que corresponde a una lista de listas. Cada sublista representa una postura diferente del robot y contiene los cinco Ć”ngulos articulares que serĆ”n enviados de forma simultĆ”nea.

Posteriormente, la instrucciĆ³n


recorre cada una de las configuraciones almacenadas. La variable `i` almacena el nĆŗmero de la configuraciĆ³n, mientras que `config` contiene la lista de Ć”ngulos correspondiente. Para cada iteraciĆ³n se llama a la funciĆ³n

Finalmente, la instrucciĆ³n

Este tiempo permite que el robot alcance completamente la posiciĆ³n deseada antes de iniciar el siguiente movimiento, evitando cambios demasiado rĆ”pidos entre posturas y garantizando una ejecuciĆ³n ordenada de la secuencia.

# Actividad 10. Trayectoria sinusoidal de una articulaciĆ³n 

Para desarrollar esta actividad se implementĆ³ la funciĆ³n `trayectoria_sinusoidal()`, cuyo objetivo es generar una trayectoria sinusoidal para una de las articulaciones del PhantomX Pincher, en este caso la articulaciĆ³n de la **cintura (base)**. La trayectoria se define mediante la ecuaciĆ³n

\[
q(t)=q_0+A\sin(2\pi ft)
\]

donde \(q_0\) corresponde a la posiciĆ³n inicial de la articulaciĆ³n, \(A\) es la amplitud de la seĆ±al sinusoidal y \(f\) es la frecuencia de oscilaciĆ³n.

La funciĆ³n recibe como parĆ”metros el nĆŗmero de la prueba (`num_prueba`), la posiciĆ³n inicial (`q0`), la amplitud (`A`), la frecuencia (`f`) y la duraciĆ³n de la prueba (`duracion_seg`). Inicialmente se define una frecuencia de muestreo de **50 Hz**, a partir de la cual se calcula el intervalo de tiempo entre muestras (`dt`) y el nĆŗmero total de iteraciones que se ejecutarĆ”n durante la prueba.

Antes de comenzar el movimiento se crean cuatro listas para almacenar el tiempo, la posiciĆ³n deseada, la posiciĆ³n medida y el error instantĆ”neo. Estos datos son utilizados posteriormente para calcular las mĆ©tricas de desempeĆ±o y generar la grĆ”fica correspondiente.

Dentro del ciclo principal se calcula el instante de tiempo actual y, utilizando la ecuaciĆ³n de la trayectoria, se obtiene la posiciĆ³n deseada para la articulaciĆ³n. Dado que la simulaciĆ³n se realiza directamente en RViz, la posiciĆ³n medida coincide con la posiciĆ³n enviada al robot, por lo que ambas toman el mismo valor durante la ejecuciĆ³n. Con estos datos se calcula el error instantĆ”neo como la diferencia absoluta entre la posiciĆ³n deseada y la medida.

Una vez calculada la posiciĆ³n deseada, esta se asigna a la primera articulaciĆ³n del vector `pose_actual_grados` y posteriormente se envĆ­a al robot mediante la funciĆ³n

```python
self.desplazar_a_configuracion(pose_actual_grados)
```

la cual fue descrita en el punto 7 y es la encargada de publicar simultĆ”neamente la configuraciĆ³n articular utilizando un mensaje de tipo `JointState`.

Al finalizar el recorrido, la funciĆ³n calcula dos indicadores de desempeĆ±o. El primero corresponde al error mĆ”ximo, obtenido como el mayor valor registrado en la lista de errores. El segundo es el error cuadrĆ”tico medio (MSE), calculado como el promedio del cuadrado de todos los errores registrados durante la trayectoria. Estas mĆ©tricas permiten cuantificar el seguimiento de la trayectoria programada.

Finalmente, utilizando los datos almacenados durante la ejecuciĆ³n, se genera una grĆ”fica en la que se compara la posiciĆ³n deseada con la posiciĆ³n medida para cada una de las pruebas realizadas, facilitando el anĆ”lisis visual del comportamiento de la articulaciĆ³n. Sin embargo, como todo se hizo en un entorno de simulaciĆ³n las grĆ”ficas resultaron en que el seguimiento fue perfecto como se muestra a continuaciĆ³n.
